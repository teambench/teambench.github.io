Architecture

Three-Role Pipeline with Container Isolation

Each role runs in an isolated Docker container. No single container can access everything.

1

Planner

Reads the full specification (spec.md). Decomposes requirements, identifies hidden constraints and edge cases, creates an execution plan. Cannot execute code or modify the workspace.

2

Executor

Reads only a brief summary (brief.md). Runs commands and modifies workspace code following the Planner's instructions. Cannot access the full specification.

3

Verifier

Reads spec.md and workspace (read-only). Checks compliance against requirements and writes attestation.json. Cannot modify the workspace or run commands.

Structural Enforcement, Not Prompt Compliance

Container bind mounts create a controlled experiment where role boundaries are physically enforced by the operating system. Without structural enforcement, performance differences cannot be attributed to genuine collaboration versus varying degrees of model compliance.

  • Information asymmetry by design — Specification contains critical details absent from the brief
  • Five ablation conditions — Systematically remove roles to isolate marginal contribution
  • Parameterized generators — 153 generators produce distinct instances from different seeds
  • Partial scoring — Deterministic graders produce scores in [0, 1]
TNI = (Steam − Srestricted) / max(ε, Soracle − Srestricted)
TNI = 1.0 means full recovery of the oracle-restricted gap; TNI > 1.0 means team exceeds oracle

Key Findings

When Does Multi-Agent Collaboration Help?

Team benefit is conditional, not universal. It concentrates on the hardest tasks and weakest models.

The Equalizer Effect

Team coordination helps most when single agents fail. The weakest model shows the largest team uplift, while the strongest see minimal gains.

+32.1% weakest model

Planning Drives Value

Planning adds +4.8% on average (p=0.02). The Planner's specification relay provides knowledge executors cannot independently access.

+4.8% planning value

Verification Overhead

Current LLM verifiers add no net value and actively hurt 31% of tasks. The most common failure mode is false rejection of valid solutions.

−5.8% verification

Hardest Tasks Benefit Most

Tasks in the hardest quintile (oracle < 22%) show +15.3% team uplift. Easy tasks show negative uplift as coordination adds overhead.

+15.3% on Q1 hardest

Universal Across Families

All six closed-source models from three provider families show positive team uplift on pass rate. Zero tasks consistently suffer across all models.

6/6 models benefit

Tool Specialization

When roles have specialized tools (static analyzers for Planner, test runners for Verifier), the team achieves 72% vs 32% oracle.

TNI = 1.25
