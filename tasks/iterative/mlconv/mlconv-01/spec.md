# MLCONV-01: Class Imbalance Discovery

## Overview

You are given a credit card fraud detection pipeline. The model achieves 0.998 accuracy on the test set but only **0.41 macro-F1**. Your task is to diagnose and fix the class imbalance problem so the model achieves **macro-F1 â‰Ą 0.80** on the held-out test set.

## Current Problem

The pipeline in `pipeline.py` trains a `RandomForestClassifier` without addressing class imbalance. The model learns to predict everything as negative:

- Accuracy: ~0.998 (misleading â€” just predicts majority class)
- Macro-F1: ~0.41 (fails because minority class recall is near zero)

## Required Fix

You must address the class imbalance. Accepted approaches:
1. Add `class_weight='balanced'` to the `RandomForestClassifier` **AND**
2. Oversample the minority class using SMOTE (from `imbalanced-learn`) before training

Both steps together are needed to reliably reach the threshold.
