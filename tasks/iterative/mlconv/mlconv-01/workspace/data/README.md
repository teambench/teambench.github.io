# Data — Credit Card Fraud Detection

## Real-World Dataset

This task is derived from the **ULB Machine Learning Group Credit Card Fraud Detection** dataset,
publicly available on Kaggle:

> https://www.kaggle.com/mlg-ulb/creditcardfraud

### Citation

Andrea Dal Pozzolo, Olivier Caelen, Reid A. Johnson and Gianluca Bontempi.
*Calibrating Probability with Undersampling for Unbalanced Classification.*
In Symposium on Computational Intelligence and Data Mining (CIDM), IEEE, 2015.

### Dataset Summary (from the paper)

| Property | Value |
|----------|-------|
| Total transactions | 284,807 |
| Fraudulent transactions | 492 |
| Fraud rate | 0.172% |
| Time span | 2 days (September 2013, European cardholders) |
| Features | V1–V28 (PCA-transformed), plus `Time` and `Amount` |
| Target column | `Class` (1 = fraud, 0 = legitimate) |

The original features are PCA-transformed to protect cardholder privacy. Only `Time` (seconds
elapsed since first transaction) and `Amount` (transaction amount in EUR) are left in their
original form.

**Key statistic:** 492 frauds out of 284,807 transactions = **0.172% fraud rate** — one of the
most extreme class imbalance ratios in publicly studied fraud datasets.

## Sample Data in This Workspace

Because the Kaggle dataset requires accepting a license agreement and cannot be redistributed
automatically, this workspace uses a **statistically faithful synthetic sample** generated by
`generate_sample.py`.

The sample preserves:
- The 0.172% fraud imbalance ratio
- 28 numeric PCA-like features (V1–V28) with distributions matching published statistics
- `Amount` column with log-normal distribution (mean ~88 EUR, skewed right)
- `Time` column spanning a 48-hour window

To regenerate the data files:

```bash
python generate_sample.py
```

This creates `train.csv` and `test.csv` in this directory.

## Differences from Original Dataset

| Property | Original Kaggle | This Sample |
|----------|-----------------|-------------|
| Size | 284,807 rows | 5,000 train / 1,000 test |
| Feature names | V1–V28 + Amount + Time | feature_0–feature_27 + Amount + Time |
| Fraud rate | 0.172% | ~0.172% (statistically matched) |
| PCA features | Real PCA projections | Synthetic with matched mean/std |
| License | CC BY-SA 4.0 (Kaggle) | No license restriction |

The benchmark task uses `feature_0`–`feature_9` (a 10-feature subset) plus `Amount` and `Time`
to keep training fast while preserving the essential imbalance challenge.
